> 디자인 기준 DS-01: 권한·개인정보 경고는 색만이 아니라 제목·설명·확인 동작을 가진 12px modal로 표시한다.

## 신뢰 경계

Spring Security가 실제 인증·인가와 리소스 권한을 강제한다. Next.js의 메뉴 숨김, redirect, Proxy는 사용자 편의와 낙관적 검사일 뿐 최종 보안이 아니다.

## 환경변수

- 공개 가능: `NEXT_PUBLIC_API_BASE_URL`
- 공개 금지: JWT 서명키, DB 정보, 스토리지 비밀키, 내부 API 키, 원본 사진 경로
- `.env*`는 저장소에 커밋하지 않고 root에서만 로드한다.
- `NEXT_PUBLIC_` 값은 build 시점에 브라우저 bundle에 고정됨을 고려한다.

## 최소 데이터 원칙

- Client Component에는 화면에 필요한 최소 DTO만 전달한다.
- 일반 사용자 참여자 응답에는 마스킹된 이름 등 공개 필드만 포함한다.
- 관리자 메모를 일반 응답으로 받은 뒤 UI에서 숨기는 방식은 금지한다.
- 서버 전용 모듈은 필요 시 `import 'server-only'`를 사용한다.

## 파일과 로그

브라우저는 확장자, MIME, 20MB, preview 가능 여부만 사전 확인한다. 실제 파일 유효성은 서버가 재검증한다. 토큰, cookie, 전화번호, 생년월일, 관리자 메모, 원본 사진 경로를 로그하지 않는다.

## 체크리스트

- [ ] Client bundle에 비밀 환경변수 접근이 없다.
- [ ] 관리자·사용자 DTO가 분리된다.
- [ ] route/search/form 입력을 신뢰하지 않고 검증한다.
- [ ] 오류 화면이 서버 내부 message/stack을 노출하지 않는다.
- [ ] 실제 개인정보 없는 Mock만 커밋한다.
